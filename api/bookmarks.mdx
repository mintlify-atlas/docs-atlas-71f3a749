## Bookmarks Endpoints

Manage user bookmarks for verses. All endpoints require authentication.

## GET /api/bookmarks

Retrieve all bookmarks for the authenticated user.

### Authentication

Requires Clerk authentication. Include authentication headers with the request.

### Response

Returns an array of bookmarked verses sorted by creation date (newest first).

<ResponseField name="[].id" type="integer">
  Unique bookmark ID
</ResponseField>

<ResponseField name="[].user_id" type="string">
  User's unique identifier
</ResponseField>

<ResponseField name="[].chapter" type="integer">
  Chapter number (1-18)
</ResponseField>

<ResponseField name="[].verse" type="integer">
  Verse number
</ResponseField>

<ResponseField name="[].translation" type="string">
  English translation of the verse
</ResponseField>

<ResponseField name="[].summarized_commentary" type="string">
  Commentary or summary for the verse
</ResponseField>

<ResponseField name="[].created_at" type="string">
  ISO 8601 timestamp when the bookmark was created
</ResponseField>

### Response Example

```json
[
  {
    "id": 123,
    "user_id": "user_abc123",
    "chapter": 2,
    "verse": 47,
    "translation": "You have the right to perform your prescribed duties...",
    "summarized_commentary": "This verse teaches the principle of performing duty...",
    "created_at": "2024-03-10T12:34:56.789Z"
  }
]
```

### Error Responses

| Status Code | Description |
|-------------|-------------|
| 401 | Unauthorized - authentication required |
| 429 | Rate limit exceeded (30 requests per minute) |
| 500 | Failed to fetch bookmarks |
| 503 | Database not configured |

## POST /api/bookmarks

Create a new bookmark for a verse.

### Authentication

Requires Clerk authentication.

### Request

<ParamField body="chapter" type="integer" required>
  Chapter number (1-18)
</ParamField>

<ParamField body="verse" type="integer" required>
  Verse number (must be valid for the chapter)
</ParamField>

<ParamField body="translation" type="string" required>
  English translation of the verse (max 10,000 characters)
</ParamField>

<ParamField body="summarized_commentary" type="string" required>
  Commentary or summary for the verse (max 10,000 characters)
</ParamField>

### Request Body Example

```json
{
  "chapter": 2,
  "verse": 47,
  "translation": "You have the right to perform your prescribed duties...",
  "summarized_commentary": "This verse teaches the principle of performing duty..."
}
```

### Response

```json
{
  "success": true
}
```

### Error Responses

| Status Code | Description |
|-------------|-------------|
| 400 | Missing required fields, invalid chapter/verse, or text too long |
| 401 | Unauthorized |
| 409 | Already bookmarked (duplicate) |
| 429 | Rate limit exceeded |
| 500 | Failed to save bookmark |
| 503 | Database not configured |

---

## DELETE /api/bookmarks

Remove a bookmark for a specific verse.

### Authentication

Requires Clerk authentication.

### Query Parameters

<ParamField query="chapter" type="integer" required>
  Chapter number (1-18)
</ParamField>

<ParamField query="verse" type="integer" required>
  Verse number
</ParamField>

### Response

```json
{
  "success": true
}
```

### Error Responses

| Status Code | Description |
|-------------|-------------|
| 400 | Invalid chapter or verse |
| 401 | Unauthorized |
| 429 | Rate limit exceeded |
| 500 | Failed to remove bookmark |
| 503 | Database not configured |

---

## Rate Limiting

All bookmark endpoints: **30 requests per minute** per client

## Code Examples

<CodeGroup>

```bash cURL - Get Bookmarks
curl -X GET https://gitachat.org/api/bookmarks \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```bash cURL - Create Bookmark
curl -X POST https://gitachat.org/api/bookmarks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "chapter": 2,
    "verse": 47,
    "translation": "You have the right to perform your prescribed duties...",
    "summarized_commentary": "This verse teaches the principle..."
  }'
```

```bash cURL - Delete Bookmark
curl -X DELETE "https://gitachat.org/api/bookmarks?chapter=2&verse=47" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
// Get bookmarks
const bookmarks = await fetch('https://gitachat.org/api/bookmarks', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
}).then(r => r.json());

// Create bookmark
await fetch('https://gitachat.org/api/bookmarks', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    chapter: 2,
    verse: 47,
    translation: 'You have the right...',
    summarized_commentary: 'This verse teaches...'
  })
});

// Delete bookmark
await fetch('https://gitachat.org/api/bookmarks?chapter=2&verse=47', {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

```typescript TypeScript
interface Bookmark {
  id: number;
  user_id: string;
  chapter: number;
  verse: number;
  translation: string;
  summarized_commentary: string;
  created_at: string;
}

// Get bookmarks
const response = await fetch('https://gitachat.org/api/bookmarks', {
  headers: { 'Authorization': `Bearer ${token}` }
});
const bookmarks: Bookmark[] = await response.json();
```

</CodeGroup>

### Notes

- Bookmarks are unique per user, chapter, and verse combination
- Attempting to bookmark the same verse twice returns a 409 conflict error
- Bookmarks are stored in Supabase database
- All text fields have a maximum length of 10,000 characters
